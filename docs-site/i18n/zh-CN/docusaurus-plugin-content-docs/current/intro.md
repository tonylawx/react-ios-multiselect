# 介绍

`react-ios-multiselect` 是一个移动优先的 React `<Select>` 组件，带有原生
iOS 风格的底部 sheet。它在同一个组件里支持单选和多选，在搜索并选择时**保持
键盘不收起**（一个出了名难搞的 iOS WebKit 行为），虚拟化渲染 2,000+ 选项仍
然流畅，并且**除 `react` / `react-dom` 外零运行时依赖**。

## 为什么需要它

大多数 React select 组件要么是只在桌面可用的 popover（在移动端体验很差），
要么是原生 `<select multiple>` 的薄封装（无法自定义样式、触屏多选体验差）。
本组件给你一个像原生 iOS picker 一样行为的 sheet，同时提供组件库级别的样式
与行为控制——而且不往你的依赖树里塞任何别的东西。

## 特性

- **iOS 原生 sheet** —— 从底部滑出、悬于键盘之上、惯性滚动，可点遮罩 / ESC /
  表头按钮关闭。
- **iOS 原生选中态** —— 选中行变 iOS 蓝 + 实心圆形对勾（蓝底白勾），不是通用
  的线框图标。
- **键盘感知** —— 搜索框聚焦时点击行，会切换选中**但不收起键盘**。
- **虚拟化** —— 只挂载可见窗口 + overscan 的行。
- **单选 + 多选同一个组件** —— 用 `multiple` prop 切换模式。
- **草稿-提交式多选** —— 切换只修改本地草稿；蓝色 ✓ 提交，✕ 丢弃。
- **对 AI agent 友好** —— 稳定的 `data-rios-*` 选择器 + 完整 ARIA + 带强制声明
  的 AI 贡献工作流。
- **零依赖** —— 纯 CSS + `--rios-*` 自定义属性做主题。
